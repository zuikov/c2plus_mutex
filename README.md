# LAB синхронизацию процессов в ОС Windows

# Програми для консольного процесу Boss і консольних процесів Parent, Child. Для моделювання передачі повідомлень введени спеціальні події,
# які позначають «А» , «B» і кінець сеансу для процесів Parent і Child.

# Процес Boss:
# 	запитує у користувача кількість процесів Parent і кількість процесів Child,
# 	які він повинен запустити;
# 	запускає задану кількість процесів Parent, Child;
# 	запитує в повідомленнях, отриманих від Parent або Child
# 	приймає від кожного процесу Parent, Child повідомлення і виводить повідомлення і хто його відправив на консоль в одному рядку. Приймати повідомлення може тільки від одного процесу Child і одного процесу Parent, передача інших повідомлень від інших процесів повинна блокуватися з допомогою м'ютексів;
# 	завершує свою роботу.

# Процес Parent:
# 	запрошувати з консолі повідомлення, що складається з «А» і передає їх (по одному) процесу Boss;
# 	завершує свою роботу.

# Процес Child:
# 	запрошувати з консолі повідомлення, що складається з «B» »і передає їх (по одному) процесу Boss;
# 	завершує свою роботу.
